Settings  
- disabled: Assumes external analog or digital volume controller is used.
- Hardware: Use audio device onboard volume controller.
- Software: Use MPD volume controller (32 bit float with dither). The "disabled" setting will result in 100% (0dB) volume output, otherwise
volume will automatically be reset to 0 after changing this setting.
Very High Quality: 95% bandwidth, 175dB noise rejection, linear phase
High Quality: 95% bandwidth, 120dB noise rejection, linear phase
Medium Quality: 95% bandwidth, 100dB noise rejection, linear phase
MPD can run SoX in multiple threads for improved efficiency.
Select "no" if mp3 playback ends abruptly.
Native DSD is used automatically if supported in Linux/ALSA and DAC.
- Yes = DoP is used if native DSD support is not available.
- No = DSD to PCM on-the-fly conversion is used if native DSD support is not available.
Adjusts the volume of all songs played so that they sound as though they are of equal loudness.
- NOTE: output bit depth will be converted to 16 bit.
MPD will adjust the volume of songs played using ReplayGain tags.
- Album uses the album ReplayGain tag
- Track uses the track ReplayGain tag
- Auto uses the track ReplayGain tag if MPD random play is activated otherwise the album ReplayGain tag is be used.
Default size is 4096 KB, min/max size is 4096/65535.
Default fill is 10%, min/max fill is 10%/30%.
Publish MPD Zeroconf service name with Zeroconf / Avahi.
